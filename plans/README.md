Generated by the improve + frontend-design skills on 2026-07-14. Execute in the order below — later plans depend on earlier foundations.

## Design Direction

The current TorqueDash UI is functional but generic: system fonts, no dark mode, no motion, broken mobile, sparse accessibility. The target is a **driver-focused telemetry dashboard** — fast to scan, pleasant in a garage or cockpit, distinctive without being decorative.

**Aesthetic direction:** Industrial-technical. Monospace accents for data, high-contrast gauges, dark-first palette with a sharp accent color (amber or teal for automotive feel). Think Motec i2 or AiM Race Studio — tools that look like they mean business.

## Findings considered and rejected

- **"Replace Tremor with shadcn/ui"**: Tremor v3 is already integrated and working. The marginal visual improvement doesn't justify the migration effort for 16 components. Plan 001 fills the design-token gap.
- **"Add Framer Motion"**: CSS transitions + @keyframes cover all needed animations for this scope. Framer Motion adds 43kB for animation variants we won't use.
- **"Migrate to Next.js"**: The app is a SPA behind an Express API. Next.js adds SSR complexity with no SEO benefit for a private telemetry tool.
